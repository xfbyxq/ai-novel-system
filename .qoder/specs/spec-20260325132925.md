# 日志统一管理实现计划

## 背景
用户要求小说系统的日志统一在 `logs` 目录下生成，需要具备过期删除（保留7天）、支持配置文件大小功能。组件范围为后端 + Workers。

## 现状分析
- 日志配置集中在 `core/logging_config.py`，使用 `RotatingFileHandler`
- 当前使用相对路径 `novel_system.log`，会生成在应用启动目录
- `logs/` 目录目前不存在
- Workers 日志没有专门配置，依赖根 logger

## 实现方案

### 1. 修改 `backend/config.py`
在 Settings 类中添加日志配置项：
- `LOG_DIR`: 日志目录（默认 "logs"）
- `LOG_FILE_MAX_BYTES`: 单个日志文件大小（默认 10MB）
- `LOG_FILE_BACKUP_COUNT`: 轮转备份数量（默认 5）
- `LOG_RETENTION_DAYS`: 日志保留天数（默认 7）
- `LOG_FILE_NAME`: 后端日志文件名（默认 "app.log"）
- `LOG_WORKER_FILE_NAME`: Worker 日志文件名（默认 "worker.log"）

### 2. 重构 `core/logging_config.py`
实现以下功能：
- 使用绝对路径（基于项目根目录）
- 分离后端和 Worker 日志到不同文件
- 启动时自动清理超过7天的过期日志
- 确保 logs 目录存在

核心函数：
- `ensure_log_directory()`: 确保日志目录存在
- `cleanup_old_logs()`: 清理过期日志文件
- `setup_logging()`: 配置后端日志
- `setup_worker_logging()`: 配置 Worker 日志

### 3. 修改 `.env`
添加日志配置：
```bash
LOG_DIR=logs
LOG_FILE_MAX_BYTES=10485760
LOG_FILE_BACKUP_COUNT=5
LOG_RETENTION_DAYS=7
LOG_FILE_NAME=app.log
LOG_WORKER_FILE_NAME=worker.log
```

### 4. 修改 `workers/celery_app.py`
在 Celery 应用初始化后调用 `setup_worker_logging()`

## 关键文件清单
| 文件 | 操作 |
|------|------|
| `backend/config.py` | 扩展配置类 |
| `core/logging_config.py` | 重构实现 |
| `.env` | 添加配置项 |
| `workers/celery_app.py` | 添加 Worker 日志初始化 |

## 验证方式
1. 启动后端应用，检查 `logs/app.log` 是否生成
2. 检查过期日志清理功能（启动时自动清理）
3. 测试日志轮转（当文件达到配置大小时）
4. 启动 Celery Worker，检查 `logs/worker.log` 是否生成
5. 调整 `.env` 中的 `LOG_FILE_MAX_BYTES` 测试配置是否生效
