# Release Plan v1.1.0

## 📅 发布日期
2026年2月28日

## 🎯 版本目标
本次发布主要集成了 AgentMesh 架构，实现了 Agent 间的智能信息共享和持久化记忆系统，显著提升了小说生成的一致性和连贯性。

## 🚀 主要新特性

### 1. AgentMesh 集成 (Agent 间信息共享)
- **NovelTeamContext 类**：实现跨章节状态跟踪和信息共享
- **上下文管理**：维护 Agent 输出历史、角色状态、时间线追踪
- **规则系统**：支持团队协作规则定义和执行

### 2. 持久化记忆系统
- **SQLite + FTS5 存储**：高性能的关系型数据库存储
- **章节摘要持久化**：自动保存每章的关键事件、角色变化、情节进展
- **角色状态跟踪**：实时记录角色的位置、修为、情绪等状态变化
- **伏笔管理系统**：智能识别和追踪故事中的伏笔元素

### 3. 智能上下文增强
- **AI 聊天集成**：在对话中融入持久化记忆上下文
- **小说分析增强**：基于历史信息提供更精准的小说分析
- **连续性检查**：利用持久化记忆进行智能连续性校验

## 📁 影响的文件

### 新增文件
- `agents/team_context.py` - TeamContext 核心实现
- `agents/foreshadowing_tracker.py` - 伏笔追踪系统
- `agents/continuity_fixer.py` - 连续性检查增强
- `backend/services/agentmesh_memory_adapter.py` - 持久化记忆适配器
- `novel_memory/novel_memory.db` - 持久化记忆数据库

### 修改文件
- `agents/agent_dispatcher.py` - 增加 team_context 支持
- `agents/crew_manager.py` - 集成 TeamContext 上下文管理
- `backend/services/generation_service.py` - 重构支持持久化记忆
- `backend/services/ai_chat_service.py` - 集成持久化记忆增强分析

## 🔧 技术改进

### 架构优化
- 引入观察者模式实现 Agent 间松耦合通信
- 采用责任链模式处理复杂的上下文流转
- 实现双写机制确保内存和持久化存储同步

### 性能提升
- FTS5 全文检索优化查询性能
- WAL 模式提升并发写入性能
- 智能缓存减少重复计算
