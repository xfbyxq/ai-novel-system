## 二、P0 级改进：逻辑一致性修复

### 2.1 新增 `GlobalConsistencyChecker`（全局一致性检查器）

**问题**：主角名字"林尘" vs "林萧"、性别代词"她" vs "他"混用。

**根因**：系统当前只在前端提示词层面要求一致性，但没有独立的、强制执行的全局校验层。`ContinuityChecker` 的检查基于 LLM，对明显的命名/代词错误可能漏检。

**方案**：新建 `agents/global_consistency_checker.py`，在章节生成后、入库前执行确定性规则检查：

```
功能模块：
├── NameConsistencyCheck  → 实体名与设定档对比，检测别名/变体
├── PronounConsistencyCheck → 基于角色性别档案，检测代词使用
├── TimelineSanityCheck   → 时间线单调递增校验
├── PowerLevelCheck       → 战力变化合理性校验
└── FactConsistencyCheck  → 关键事实（死亡/物品状态/地点）一致性
```

**关键设计**：

1. **实体注册表（Entity Registry）**：从角色档案、世界观设定中提取所有实体的标准化信息（名字、性别、境界、位置等），写入 JSON 格式的注册表。
2. **规则引擎**：使用正则 + 关键词匹配进行确定性检测（不依赖 LLM），避免"检查者自己也犯错"的问题。
3. **自动修复建议**：发现问题后生成明确的修复指令，注入修订流程。

**集成点**：在 `generation_service.py` 中，章节通过 Editor 审查后、入库前，新增 `GlobalConsistencyChecker` 调用。

```python
# 伪代码：集成点
async def generate_chapter(...):
    # ... 现有流程：Writer 生成 → Editor 审查 ...
    
    # 【新增】全局一致性检查
    consistency_result = await global_checker.check(
        content=final_content,
        chapter_number=chapter_num,
        entity_registry=entity_registry,
        timeline_state=timeline_tracker.get_state(),
    )
    if not consistency_result.passed:
        # 注入修复指令，触发额外修订轮
        fix_prompt = consistency_result.build_fix_prompt()
        final_content = await _apply_fixes(final_content, fix_prompt)
    
    # 入库
    await save_chapter(...)
```

### 2.2 增强 `TimelineTracker` 时间线强制校验

**问题**：第2章结尾深夜 → 第3章深夜 → 第4章又"夜风如刀"，时间推进混乱。

**当前状态**：`timeline_tracker.py` 存在且有 37KB 代码，但报告表明时间校验未生效。

**改进**：

1. **新增 `enforce_monotonic_time()` 方法**：每章结束后记录时间标记，下一章的时间标记必须 >= 上一章。如检测到时间回退，立即标记为 high severity 问题。
2. **时间标记标准化**：要求章节策划阶段输出明确的时间标签（`"day_1_night"`, `"day_2_morning"`），而非模糊描述。
3. **在 `ChapterPlanner` 提示词中追加时间约束**：

```
【强制时间约束】
- 上一章时间标记：{previous_time_mark}
- 本章时间标记必须晚于或等于上一章，禁止时间回退
- 时间跨度必须在章节计划中明确标注（如："从深夜到次日清晨"）
```

### 2.3 新增战力合理性校验（Power Level Validator）

**问题**：淬火境初期秒杀十余人，战力膨胀过快。

**方案**：在 `character_consistency_tracker.py` 中新增战力变化追踪：

1. **境界-能力映射表**：定义每个境界的理论能力上限（如"淬火境初期：可对1-2名同境对手有优势"）。
2. **战斗结果校验**：检测章节中的战斗结果是否超出主角当前境界的合理范围。
3. **成长冷却**：设定境界提升的最小章节间隔（如至少 5 章才能提升一个小境界）。

## 三、P1 级改进：角色与文笔优化

### 3.1 增强 `CharacterConsistencyTracker` — 情感维度追踪

**问题**：林萧6章只有"冷静"一种情绪，小翠全程"害怕→哭泣"。

**当前状态**：`character_state_extractor.py` 仅提取 `emotional_state` 简单字符串，无情感深度分析。

**改进**：

1. **扩展角色档案**：在角色模型中增加 `emotional_range` 字段，定义该角色应具备的多种情感表现。例如：
   ```json
   {
     "name": "林萧",
     "emotional_range": ["冷静", "愤怒", "温柔", "幽默", "犹豫", "悲伤"],
     "dominant_emotion": "冷静",
     "min_emotion_variety_per_chapters": 3
   }
   ```

2. **新增 `EmotionDiversityChecker`**：在审查循环中检查主角在最近 N 章中是否展现了足够的情感多样性。如果只有单一情感，触发修订。

3. **在 Writer 提示词中注入角色情感指令**：
   ```
   【角色情感要求 - 林萧】
   - 本章必须展现以下情感中的至少一种：愤怒/幽默/温柔/犹豫
   - 禁止只使用"冷静"作为唯一情感状态
   - 对小翠的场景中必须展现温柔或保护欲
   ```

### 3.2 小翠主动行为强制检测

**问题**：小翠6章中"有独立行动章节 = 0"。

**方案**：

1. **在 `ChapterPlanner` 提示词中增加配角参与约束**：
   ```
   【配角参与要求】
   - 小翠在本章必须有至少一次主动行为（非被动反应）
   - 主动行为包括：主动提问、主动建议、主动行动、表达独立想法
   ```

2. **在 Editor 审查维度中新增 `character_agency` 评分项**：评估每个重要配角是否有主动行为。

### 3.3 新增 `LexicalDiversityChecker`（词汇多样性检测器）

**问题**："瞳孔微缩"出现8次，"指节泛白"5次，句式"林萧XX，目光XX"高度模板化。

**方案**：新建 `agents/lexical_diversity_checker.py`：

```
功能：
├── PhraseFrequencyAnalyzer  → 统计短语在连续章节中的出现频率
├── PatternDetector          → 检测固定句式模板（正则匹配）
├── SynonymSuggester         → 提供同义词替换建议
└── DiversityScoreCalculator → 计算词汇多样性评分
```

**高频词黑名单**（基于报告提取）：

| 短语 | 当前出现次数 | 触发阈值 | 替代方案 |
|------|------------|---------|---------|
| 瞳孔微缩 | 8 | 2次/10章 | 眼神一凝、目光骤冷、眸光微动、双目微眯 |
| 指节泛白 | 5 | 2次/10章 | 手指攥紧、握拳用力、骨节微微绷紧 |
| 摇摇欲坠的木门 | 4 | 1次/10章 | 吱呀作响的木门、斑驳的旧木门、破旧的木门 |
| 古井（形容眼神） | 4 | 2次/10章 | 深潭、寒星、幽暗的眼眸 |
| 拉风箱（呼吸） | 3 | 2次/10章 | 粗重的喘息、气息急促、呼吸粗重 |

**集成方式**：在 Editor 审查中作为自动化检查项运行，发现超阈值词汇时直接在 issues 中标注。

### 3.4 风格定位强制对齐

**问题**：简介标注"轻松幽默"，但6章正文完全是严肃复仇风格。

**方案**：

1. **在 `PromptManager.WRITER_SYSTEM` 中追加风格约束**：
   ```
   【风格定位 — 轻松幽默】
   - 每章至少包含1处幽默元素（自嘲/吐槽/反差笑点/俏皮对话）
   - 幽默需符合角色性格，不可突兀
   - 严肃场景中可用黑色幽默或冷幽默调节
   - 参考风格：主角内心独白可带自嘲，配角可有诙谐对话
   ```

2. **在 Editor 审查维度中新增 `style_consistency` 检查项**：
   - 检测章节整体风格是否与设定的"轻松幽默"一致
   - 如果连续3章无幽默元素，标记为 medium severity 问题

3. **新增 `StyleConsistencyChecker`**：基于关键词和情感分析检测风格偏差。

## 四、P1 级改进：节奏与结构优化

### 4.1 新增 `ChapterRhythmPlanner`（章节节奏规划器）

**问题**：6章中4章包含战斗，节奏全部在高紧张区域，缺乏变化。

**当前报告中的节奏分布**：
```
第1章：████████░░ 紧张
第2章：█████████░ 紧张
第3章：██████████ 极紧张
第4章：█████████░ 紧张
第5章：██████████ 极紧张
第6章：███████░░░ 中等
```

**方案**：

1. **定义章节类型枚举和节奏模板**：

```python
class ChapterType(Enum):
    BATTLE = "战斗"          # 高紧张，动作驱动
    DAILY = "日常"          # 低紧张，情感/互动驱动
    TRAINING = "修炼"       # 中紧张，成长驱动
    EXPLORATION = "探索"    # 中紧张，世界观驱动
    PLOT_TWIST = "转折"     # 高紧张，信息驱动
    EMOTIONAL = "情感"      # 中紧张，关系驱动
    SETUP = "铺垫"          # 低紧张，伏笔驱动
```

2. **节奏规划规则**：
   - 连续2章 `BATTLE` 类型后，下一章必须为非战斗类型
   - 每5章至少包含1章 `DAILY` 或 `EMOTIONAL` 类型
   - 紧张度曲线遵循波浪形：高→中→低→中→高

3. **在 `crew_manager.py` 的章节策划阶段调用节奏规划器**，为每章分配类型标签，注入 `ChapterPlanner` 提示词。

### 4.2 增强 `SimilarityDetector` 情节结构检测

**当前状态**：`similarity_detector.py` 已有 566 行，支持情节向量检测，但情节向量的提取基于关键词匹配，准确度有限。

**改进**：

1. **将 `PlotVector` 的提取从规则升级为 LLM 调用**：让 LLM 分析章节的情节结构（目标→冲突→解决→结果→情感弧线），生成更准确的情节向量。

2. **新增"情节模式重复检测"**：针对报告中的"敌人来袭→MC震慑→秒杀→发现线索"固定模式，定义专用的模式匹配规则：

```python
BATTLE_PATTERNS = [
    "敌人来袭", "主角应对", "快速击败", "发现线索"
]

# 检测连续章节是否都命中此模式
# 如连续3章命中，标记为"情节模式重复"
```

3. **降低总体相似度阈值**：将 `DUPLICATE_THRESHOLD` 从 0.30 降至 0.25，对情节结构相似度更敏感。

### 4.3 支线情节强制注入

**问题**：6章全是主线推进，无支线缓冲。

**方案**：

1. **在 `PromptManager.PLOT_ARCHITECT_TASK` 中要求至少设计3条支线**，并在卷规划中标注每条支线的触发章节。

2. **在 `ChapterPlanner` 提示词中追加支线提醒**：
   ```
   {subplot_reminder}
   <!-- 当当前章节适合插入支线时注入 -->
   <!-- 例如：本章可插入"小翠身世探索"支线 / "与XX角色的互动"支线 -->
   ```

3. **在 `continuity_integration_module.py` 中新增 `subplot_tracker`**，追踪各支线的使用频率，超过 N 章未出现的支线主动提醒插入。

## 五、P2 级改进：世界观与情感深化

### 5.1 世界观信息自然融入引导

**问题**：大量世界观信息通过"铁片激活"、"玉简阅读"一次性灌输。

**方案**：

1. **在 `WRITER_SYSTEM` 提示词中新增设定展示规则**：
   ```
   【设定展示原则 — Show, Don't Tell】
   - 世界观设定应通过角色对话、行动、冲突来展现，而非大段说明
   - 每次引入新设定时，单次说明不超过100字
   - 优先通过"角色对设定的不同理解/争论"来自然展现
   - 禁止使用"XXX是一种..."的百科式说明段落
   ```

2. **在 Editor 审查中新增"信息密度检测"**：对连续大段设定说明的章节扣分。

### 5.2 情感高潮强制检测

**问题**：6章中没有真正打动人心的情感场景。

**方案**：

1. **在 `ChapterPlanner` 提示词中强化情感高潮要求**：
   ```
   【情感高潮要求】
   - 每3章至少需要1个情感高潮点
   - 情感高潮定义：角色关系发生重大变化 / 内心冲突爆发 / 温情时刻 / 信念动摇
   - 情感高潮场景需要：
     a) 至少200字的心理/情感描写
     b) 环境氛围与情感呼应
     c) 对话推动关系变化
   ```

2. **在 Editor 审查维度中新增 `emotional_impact` 评分项**。

### 5.3 待完善：CoherenceScorecardBuilder 对接新检查器

**当前问题**：`CoherenceScorecardBuilder` 仍从旧的审查结果中提取分数，尚未对接新的质量检查器。

**改进方案**：在 `build_scorecard()` 方法中新增参数，接受新检查器报告：

```python
def build_scorecard(
    self,
    chapter_number: int,
    # ... 原有参数 ...
    # 【新增】新检查器报告
    global_consistency_report: Optional[Dict] = None,
    lexical_diversity_report: Optional[Dict] = None,
    emotion_diversity_report: Optional[Dict] = None,
    style_consistency_report: Optional[Dict] = None,
) -> CoherenceScorecard:
    # 从新检查器报告中提取分数覆盖默认值
    if global_consistency_report:
        # 使用 GlobalConsistencyChecker.overall_score 覆盖
        ...
    if style_consistency_report:
        # 使用 StyleConsistencyChecker.style_match_score 映射到 0-10
        ...
```

**分数映射规则**：

| 检查器 | 原始分数 | 映射到维度 | 映射方式 |
|--------|---------|-----------|---------|
| `GlobalConsistencyChecker` | `overall_score` (0-10) | `character_consistency` | 直接使用 |
| `LexicalDiversityChecker` | `diversity_score` (0-10) | `plot_coherence` | 直接使用（文笔影响叙事） |
| `EmotionDiversityChecker` | `diversity_score` (0-10) | `character_consistency` | 与 GlobalConsistencyChecker 取加权平均 |
| `StyleConsistencyChecker` | `style_match_score` (0-1) | `style_consistency` | × 10 映射到 0-10 |

### 6.3 生成流程改造

**改造前**：
```
ChapterPlanner → Writer → Editor Review → Save
```

**改造后（已实现）**：
```
ChapterRhythmPlanner → ChapterPlanner → Writer → Editor Review
    → GlobalConsistencyChecker → LexicalDiversityChecker
    → EmotionDiversityChecker → StyleConsistencyChecker
    → SubplotTracker → ChapterRhythmPlanner(验证)
    → (如检测到 high severity 问题) → 质量修订师 → Save
```

**流程细节**：

1. **前置规划阶段**（`crew_manager.py`）：
   - `ChapterRhythmPlanner.plan_chapter()` 分配章节类型
   - `SubplotTracker.check_and_remind()` 生成支线提醒
   - 节奏规划和支线提醒注入 ChapterPlanner 提示词

2. **审查循环阶段**（`review_loop.py`）：
   - Editor 7维度评分（含新增 `style_consistency`、`character_agency`）
   - 情感多样性检查在审查循环中注入 Writer 提示词

## 九、生成流程实际执行路径（已实现）

```
ChapterRhythmPlanner  ──→  分配章节类型（战斗/日常/修炼/探索/转折/情感/铺垫）
     ↓
ChapterPlanner        ──→  接收节奏规划注入，生成章节计划
     ↓
Writer Agent          ──→  根据章节计划和节奏规划写作
     ↓
Editor Review Loop    ──→  7维度评分 + 词汇/情感/风格检查注入
     ↓
═══════════════════════════════════════════════════════════════
【Editor 审查通过 → 入库前质量门控】
═══════════════════════════════════════════════════════════════
     ↓
1. GlobalConsistencyChecker  ──→  名字/代词/时间线/事实/战力
     ↓
2. LexicalDiversityChecker   ──→  短语重复/句式模板化/词汇多样性
     ↓
3. EmotionDiversityChecker   ──→  角色情感多样性（主角/配角）
     ↓
4. StyleConsistencyChecker   ──→  风格匹配"轻松幽默"
     ↓
5. SubplotTracker            ──→  支线出现频率/插入提醒
     ↓
6. ChapterRhythmPlanner      ──→  事后验证实际类型是否符合规划
     ↓
【如检测到 high severity 问题】
     ↓
质量修订师（低温度 LLM）──→  精确修复问题，保持故事结构不变
     ↓
入库（chapter.continuity_issues 合并所有问题）
```

## 十二、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 新增检查层增加 LLM 调用次数 | 成本上升约 20-30% | 规则检测不依赖 LLM，仅修订轮次可能增加 |
| 节奏约束可能导致章节质量下降 | 日常章节可能变得平淡 | 为日常章节定制专门的 Writer 提示词模板 |
| 词汇检测可能过于严格 | 误判合理重复 | 设置白名单（如专有名词、功法名称） |
| 实施周期较长 | 延迟后续章节生成 | 分阶段部署，P0 优先 |
| **新检查器未生效但无感知** | 问题未被检测但不报错 | 已添加详细日志，每次检查都有 `logger.info/warning` 输出 |
| **质量修订可能破坏原文** | 修订后的内容风格不一致 | 使用低温度（0.3）确保精确修订，仅修复具体问题 |
| **情感检测依赖关键词** | 隐晦情感可能漏检 | 当前阶段用规则检测可接受，后续可升级为 LLM 检测 |
| **节奏规划可能与大纲冲突** | 规划的日常章节与大纲中的战斗不符 | `plan_chapter()` 接受 `suggested_type` 参数，优先尊重大纲 |
