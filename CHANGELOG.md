# Changelog

## [1.3.1] - 2026-03-19

### Added
- **JSON解析鲁棒性增强**：改进 _extract_json_from_response、_find_json_by_brackets 和 _extract_fields_manually 方法的错误处理
- **代码质量提升**：增强异常处理和输入验证机制

### Fixed
- **Critical Bug修复**：修复了 LLM 响应解析中的异常传播问题
- **数据库迁移修复**：解决 Alembic 多头部修订版本问题

## [1.2.0] - 2026-03-02

### Added
- **审查循环框架**：引入基于模板方法模式的审查循环基础框架
- **统一基础模块**：新增 agents/base目录，包含质量报告和审查结果数据结构
- **多策略JSON提取**：增强LLM响应解析能力，支持多种提取策略
- **质量评估系统**：实现多阶段质量评估与迭代控制机制
- **连续性增强模块**：改进章节连续性检查和内容相似度检测

### Changed
- **智能体架构优化**：重构审查循环处理器，支持多种具体审查处理器扩展
- **任务调度改进**：修正任务依赖检查逻辑，添加详细警告日志
- **团队上下文管理**：增强 NovelCrewManager 中的团队上下文处理能力

### Fixed
- **调度器逻辑**：修复 agent_scheduler.py 中的任务依赖检查问题
- **审查循环集成**：优化 character_review_loop、plot_review_loop、world_review_loop 的实现

## [1.1.0] - 2026-02-28

### Added
- **AgentMesh 集成**：实现 Agent 间信息共享与持久化记忆系统
- **TeamContext 上下文管理**：新增 NovelTeamContext 类，支持跨章节状态跟踪
- **持久化记忆存储**：集成 SQLite + FTS5 存储系统，支持章节摘要、角色状态、伏笔追踪
- **智能上下文增强**：AI 聊天和小说分析中集成持久化记忆上下文
- **角色状态管理**：实时跟踪角色在不同章节中的状态变化
- **伏笔追踪系统**：自动识别和管理故事情节中的伏笔元素
- **连续性检查增强**：基于持久化记忆的智能连续性校验

### Changed
- 优化 GenerationService 架构，支持持久化记忆适配器
- 改进 NovelCrewManager，集成 TeamContext 进行上下文管理
- 增强 AgentDispatcher，支持 team_context 参数传递
- 重构 AI 聊天服务，集成持久化记忆增强分析能力

### Fixed
- 修复角色状态初始化和格式化处理问题
- 优化持久化记忆系统的异常处理和日志记录
